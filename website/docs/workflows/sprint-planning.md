# Sprint Planning Workflow

Step-by-step guide for planning and running sprints.

## Before Sprint

### 1. Backlog Grooming (Day before planning)

1. Product Owner reviews backlog
2. Prioritize items by business value
3. Ensure stories have acceptance criteria
4. Estimate unestimated items
5. Mark items ready for sprint

### 2. Sprint Planning Meeting

**Duration:** 1-2 hours for a 2-week sprint

**Agenda:**

1. Review previous sprint metrics
2. Set sprint goal
3. Select items from backlog
4. Break stories into tasks
5. Assign owners
6. Commit as a team

## During Sprint

### Daily Standup

```
What did I do yesterday?
What will I do today?
Any blockers?
```

**In Gauzy:**

1. Check **Dashboard** → **Team Activity**
2. Review [Daily Plans](../features/daily-plans-deep-dive)
3. Update task statuses on the board

### Mid-Sprint Review

1. Check burndown chart
2. Flag at-risk items
3. Re-prioritize if needed
4. Unblock team members

## Sprint End

### Sprint Review

1. Demo completed work
2. Stakeholder feedback
3. Accept/reject stories
4. Update product backlog

### Retrospective

| Question            | Format           |
| ------------------- | ---------------- |
| What went well?     | Sticky notes     |
| What could improve? | Group discussion |
| Action items        | Assigned tasks   |

### Close Sprint in Gauzy

1. Go to **Projects** → **Sprints**
2. Click **Close Sprint**
3. Carry over incomplete items to next sprint
4. Review velocity metrics
